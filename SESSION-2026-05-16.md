### Phase 2: MemoryClient API — IQRA (L2)
Created 7 Next.js API route files under `src/app/api/memory/`:

1. **`GET /api/memory`** — System status & all stats
   - Bridge stats (hot/warm/cold hit rates, sizes)
   - MicroMemory stats (patterns, experiences, rewards)
   - Pulse369 stats (counter, last promote/archive/purge)
   - Redis availability check

2. **`POST /api/memory/write`** — Write to MemoryBridge
   - Parameters: `key`, `value`, `layer` (hot/warm/cold), `ttl_ms`, `broadcast`
   - Returns entry `id`
   - Triggers Pulse369 tick

3. **`POST /api/memory/read`** — Read from MemoryBridge
   - Parameters: `key`
   - Returns `{ value, layer, hit, latency_ms }`

4. **`POST /api/memory/pattern`** — Pattern operations
   - `action: search` — Similar pattern search with embedding or text, topK, minResonance
   - `action: store` — Store new pattern (verse, field, resonanceScore, embedding)
   - `action: stats` — MicroMemory stats

5. **`POST /api/memory/quantum`** — Quantum memory operations
   - `action: search` — Quantum memory search with query and optional targetConcept
   - `action: store` — Store quantum memory (content, concept, superposition, entangledWith)

6. **`POST /api/memory/topology`** — Multi-layer memory operations
   - `action: search` — Search all 7 memory layers (working, semantic, episodic, topological, graph)
   - `action: stats` — MemoryTopology stats
   - `action: write` — Write to specific type
   - `action: read` — Read from specific type

7. **`GET /api/memory/pulse`** — Pulse369 tick counter + stats
